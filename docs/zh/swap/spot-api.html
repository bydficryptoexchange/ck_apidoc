# Market Interface

# 1. Market Pair Information

HTTP Request


    # Request
    GET /cgk_rank/pairs
    example： https://www.bydfi.com/b2b/cgk_rank/pairs
    description: market data for all tickers and all markets.

response description

Name Type Status Description
success code Mandatory Status code true means success and other means failure
result String Mandatory Prompt information
ticker_id string Mandatory Identifier of a ticker with delimiter to separate base/quote, eg. BTC_USDT (Price of BTC is quoted in USD)
base string Mandatory Symbol/currency code of base pair, eg. BTC
target string Mandatory Symbol/currency code of quote pair, eg. ETH 
    # Response
    {
        "success": true,
        "result":[
                    {
               "ticker_id":"MAT-USDT",
               "base":"MAT",
               "target":"USDT",
               }
           ]
     }

# 2. Get Market Depth

HTTP Request

                    
    # Request
    GET /cgk_rank/orderbook
    example： https://www.bydfi.com/b2b/cgk_rank/orderbook
    description: full depth returned for a given market pair

request Paramters

name Type Status description
ticker_id string Mandatory A pair such as “LTC_BTC”
depth string Recommended Orders depth quantity: [0,5,10,20,50,100,500]. Default depth=100

Response description

name Type Status description
success code Mandatory Status code true means success and other means failure
ticker_id string Mandatory A pair such as "LTC_BTC", with delimiter between different cryptoassets
timestamp Integer Mandatory Unix timestamp in milliseconds for when the last updated time occurred.
asks decimal Mandatory An array containing 2 elements. The offer price and quantity fyor each bid order
bids decimal Mandatory An array containing 2 elements. The ask price and quantity for each ask order 
    # Response
     {
        "success": true,
        "result":{
               "asks":[{
               "amount":"0.08436",
               "price":"57137.3",
               ]},
               "bids":[{
               "amount":"0.08436",
               "price":"57137.3",
               ]}
           }
     }

# 3. 24-hour pricing for each market pair

HTTP Request

                    
    # Request
    GET /cgk_rank/tickers
    example： https://www.bydfi.com/b2b/cgk_rank/tickers
    description: The /tickers endpoint provides 24-hour pricing and volume information on each market pair available on an exchange

Response description

name Type Status description
ticker_id string Mandatory Identifier of a ticker with delimiter to separate base/target, eg. BTC_ETH
base_currency Integer Mandatory Symbol/currency code of base pair, eg. BTC
target_currency decimal Mandatory Symbol/currency code of target pair, eg. USDT
last_price decimal Mandatory Last transacted price of base currency based on given target currency
base_volume decimal Mandatory 24 hour trading volume in base pair volume
target_volume decimal Mandatory 24 hour trading volume in target pair volume
bid decimal Mandatory Current highest bid price
ask decimal Mandatory Current lowest ask price
high decimal Mandatory Rolling 24-hours highest transaction price
low decimal Mandatory Rolling 24-hours lowest transaction price 
    # Response
    {
        "success": true,
        "result":[
            {
               "ticker_id":"BTC_USDT",
               "base_currency":"BTC",
               "target_currency":USDT,
               "last_price":"47115.0",
               "base_volume":"47115.0",
               "target_volume":"455555",
               "bid":47115.0,
               "ask":"47115.0",
               "high":"47115.0",
               "low":"47115.0",
               }
           ]
     }

# 4. Get history order

HTTP Request

                    
    # Request
    GET /cgk_rank/historical
    example： https://www.bydfi.com/b2b/cgk_rank/historical
    description:The trades endpoint is to return data on all recently completed trades for a given market pair

request description

name Type Status description
ticker_id string Mandatory A pair such as LTC_BTC.
type string Mandatory To indicate nature of trade - buy/sell default all trade hitstory
limit string Mandatory Number of historical trades to retrieve from time of query. [0, 200, 500]. 0 returns full history.

response description

name Type Status description
trade_id integer Mandatory A unique ID associated with the trade for the currency pair transactio
price decimal Mandatory Last transacted price of base currency based on given quote currency
base_volume decimal Mandatory Transaction amount in BASE currency.
target_volume decimal Mandatory Transaction amount in QUOTE currency
trade_timestamp decimal Mandatory Unix timestamp in milliseconds for when the transaction occurred.
type string Mandatory Used to determine whether or not the transaction originated as a buy or sell
